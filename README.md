# Codificador e decodificador de RSA

Um projeto em pyhon focado em codificação e decodificação de textos para a criptografia RSA.

AVISO IMPORTANTE:
Este projeto implementa a criptografia RSA apenas com fins meramente educacionais.
Ele não utiliza números primos grandes, não implementa padding (como OAEP ou PKC#1 v1.5), permite a entrada manual de algumas chaves e usa criptografia determinística, ou seja, o mesmo texto
vai sempre apresentar o mesmo resultado.
Por tais razões, não se deve utilizá-lo em sistemas legítimos de segurança, pois é vulneravel a ataques de fatoração, chosen-plaintext e diversas outras formas de exploração.
O propósito do projeto é demonstrar a matemática e a lógica interna da criptografia, não fornecer um codificador realmente seguro.

# RSA coder and decoder

A python project focused on coding and decoding texts in the RSA cryptography

IMPORTANT NOTICE
This project implements the RSA cryptography only with merely educational purposes.
It doesn't use big prime numbers, doesn't implement padding (with OAEP or PKC#1 v1.5), allows the manual input of some keys and uses deterministic cryptography, in other words, the same text
will always show the same result.
Due to those reasons, it should NOT be used in legitimate security systems, because it's vulnerable to factorization attacks, chosen-plaintext and many other exploitation forms.
The project's purpose is to demonstrate the cryptography's math and internal logic , not supplying an actually secure coder

# RSA Encoder und Decoder

Ein python Projekt, das sich auf Kodierung und Dekodierung von Texten für die RSA-Kryptographie konzentriert

WICHTIGE WARNUNG:
Dieses Projekt setzt die RSA-Kryptographie ausschließlich zu Bildungszwecken um.
Es benutzt keine große Primzahlen, setzt kein Padding (mit OAEP oder PKC#1 v.1.5) um, erlaubt manuellen Einsätze für einige Keys und benutzt deterministische Kryptographie, also, der selbe Text
wird immer das selbe Ergebnis vorzeigen.
Für solche Gründe, es soll NICHT in rechtmäßige Sicherheitssysteme benutzt werden, da es anfällig ist zu Faktorisierungangriffe, chosen-plaintext und viele andere Ausnutzungen ist.
Die Absicht des Projektes ist die Methematik und Binnenlogik der Kryptographie und nicht einen wirklichen sicheren Encoder anliefern.
